#### Parsing the token

We need to parse the token to get the `email` field from the token. This is the unique identifier
for the user. This is dependent on the `verifierId` field you've set on the dashboard. In our case,
it's `email`. The code for parsing the `idToken` is as follows:

```tsx
import { safeatob } from "@toruslabs/openlogin-utils";

const parseToken = (token: any) => {
  try {
    const token = token.split(".")[1]; // payload
    return JSON.parse(safeatob(token));
  } catch (err) {
    console.error(err);
    return null;
  }
};

const { email } = parseToken(idToken);
```
